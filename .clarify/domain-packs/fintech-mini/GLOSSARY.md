# Glossary: Fintech

| Term | Definition |
| --- | --- |
| Ledger | The authoritative double-entry record of all balance movements. |
| Transaction | A balance-affecting event, posted as debit/credit entries. |
| Hold | Funds reserved but not yet settled. |
| Settlement | Final transfer of funds between parties. |
| Reconciliation | Matching internal ledger against provider/bank records. |
| KYC | Know Your Customer identity verification. |
| AML | Anti-Money-Laundering monitoring. |
| Idempotency key | Token ensuring a money operation runs at most once. |
| Savings deposit | An interest-bearing account where the customer deposits funds. |
| Term deposit | A deposit locked for a fixed tenor with a maturity date. |
| Interest accrual | Daily calculation of earned-but-unposted interest. |
| Interest posting / capitalization | Periodic posting of accrued interest to the balance. |
| Effective date | The date from which a rate/fee/rule version applies. |
| Grandfathering | Keeping existing entities on their prior rule/rate after a change. |
| Rate Config | The effective-dated, versioned store of interest rates and fees. |
